## 一、标签嵌套变量冲突

### 问题

EYOUCMS 所有循环标签默认使用 `{$field.*}` 变量，嵌套时内层覆盖外层，导致外层数据失效。

> 详见 [标签规划.md - 五](标签规划.md#五标签嵌套注意事项)

### 核心规则

**嵌套时内层必须定义 `id` 属性**，变量改为 `{$自定义.*}`。

### 错误 vs 正确

```html
<!-- 错误 -->
{eyou:type typeid='1'}
  <h3>{$field.typename}</h3>
  {eyou:artlist typeid='1'}         <!-- 内层也用 $field，被覆盖 -->
    <a href='{$field.arcurl}'>{$field.title}</a>
  {/eyou:artlist}
{/eyou:type}
```

```html
<!-- 正确 -->
{eyou:type typeid='1'}
  <h3>{$field.typename}</h3>
  {eyou:artlist typeid='1' id='product'}  <!-- 自定义 id -->
    <a href='{$product.arcurl}'>{$product.title}</a>
  {/eyou:artlist}
{/eyou:type}
```

### 支持 id 的循环标签

`{eyou:type}`、`{eyou:artlist}`、`{eyou:list}`、`{eyou:adv}`、`{eyou:links}`、`{eyou:tags}`、`{eyou:arcview}`、`{eyou:models}`、`{eyou:beafter}`、`{eyou:relevarticle}`、`{eyou:guestbookform}`

> 完整说明：[tags/artlist.md](tags/artlist.md) | [tags/type.md](tags/type.md) | [tags/models.md](tags/models.md)

## 二、sppurchase 标签错误

`{eyou:sppurchase /}` 是自闭合写法，该标签**必须使用完整结构**。

> 详见 [tags/sppurchase.md](tags/sppurchase.md)

### 示例

```html
<!-- 错误：自闭合不生效 -->
{eyou:sppurchase /}
```

```html
<!-- 正确：完整结构 -->
{eyou:sppurchase id='field'}
  <div class="ey-number">
    <button {$field.ReduceQuantity}>-</button>
    <input type="text" {$field.UpdateQuantity} />
    <button {$field.IncreaseQuantity}>+</button>
  </div>
  <a href="javascript:void(0);" {$field.ShopAddCart}>加入购物车</a>
  <a href="javascript:void(0);" {$field.BuyNow}>立即购买</a>
  {$field.hidden}
{/eyou:sppurchase}
```

## 三、nav.htm 会员入口错误

`{eyou:user type='userinfo'}` 不可单独使用，必须配合外层 `{eyou:user type='open'}` 包裹。

> 详见 [tags/usertag.md](tags/usertag.md) | [tags/diyurl.md](tags/diyurl.md)

### 示例

```html
<!-- 错误：单独使用 type='userinfo' 不生效 -->
{eyou:user type='userinfo'}
  <a href="{$field.url}">{$field.username}</a>
{/eyou:user}
{eyou:empty name="$eyou.user"}
  <a href="{:url('user/Users/login')}">登录</a>
{/eyou:empty}
```

```html
<!-- 正确：外层 open + 内层 userinfo -->
{eyou:user type='open'}
  {eyou:user type='userinfo' id='field'}
    <div id="{$field.htmlid}">
      <a href="{eyou:diyurl type='login' /}">登录</a>
    </div>
    {$field.hidden}
  {/eyou:user}
{/eyou:user}
```

## 四、字段变量区分

### 两种字段变量

| 变量 | 来源 | 作用域 |
| -- | -- | -- |
| `{$eyou.field.*}` | 当前页面文档数据 | 内容页全局 |
| `{$field.*}` | 当前循环的当前行 | 循环体内 |

> 详见 [tags/field.md](tags/field.md)

### 常见错误

```html
<!-- 错误：循环外使用 $field（空值） -->
{eyou:artlist loop='5' id='item'}
  <a href='{$item.arcurl}'>{$item.title}</a>
{/eyou:artlist}
{$field.title}   <!-- 循环外无 $field -->
```

```html
<!-- 正确：内容页用 $eyou.field，循环用自定义 id -->
<h1>{$eyou.field.title}</h1>
{eyou:artlist loop='5' id='item'}
  <a href='{$item.arcurl}'>{$item.title}</a>
{/eyou:artlist}
```

### 禁止使用的字段

| 字段 | 原因 | 替代 |
| -- | -- | -- |
| `{$eyou.field.artnum}` | 商品文档中不存在 | 移除 |
| `{$eyou.field.add_time}` | 未格式化 | `{$eyou.field.add_time\|MyDate='Y-m-d',###}` |

> 详见 [tags/functions.md](tags/functions.md) | [dict/ey_archives.md](dict/ey_archives.md)

## 五、URL 生成方式

前台模板中的所有链接统一使用 `{eyou:diyurl}`，仅在 JS 逻辑或 PHP 标签内使用 `{:url()}`。

> 详见 [tags/diyurl.md](tags/diyurl.md)

### 方式对比

```html
<!-- 错误：url() 在 HTML 属性中可能解析失败 -->
<a href="{:url('user/Users/login')}">登录</a>
```

```html
<!-- 正确：前台模板统一用 diyurl -->
<a href="{eyou:diyurl type='login' /}">登录</a>
```

## 七、URL 参数拼接：? vs &

### 问题

列表页排序链接使用 `?orderby=xxx` 会导致 URL 参数丢失（tid、现有参数被覆盖）。

### 原因

EYOUCMS 列表页 URL 通常已带完整参数：

```
/index.php?m=home&c=Lists&a=index&tid=1&orderby=sales_num
```

| 写法 | 结果 | 是否正确 |
|------|------|:--------:|
| `?orderby=add_time` | `?orderby=add_time` | ❌ 参数丢失 |
| `&orderby=add_time` | `&tid=1&orderby=sales_num&orderby=add_time` | ❌ 重复参数 |
| `{$currentUrl}&orderby=add_time` | 完整保留 + 追加 | ✅ |

### 正确做法

排序链接应基于当前完整 URL 追加参数，而不是用 `?` 覆盖。

```html
<!-- 错误 -->
<a href="?orderby=add_time">最新</a>

<!-- 正确：动态获取当前 URL 后追加 -->
{eyou:php}echo '?'.$_SERVER['QUERY_STRING'].'&orderby=add_time';{/eyou:php}
```

### lists_product.htm 修正示例

```html
<!-- 筛选栏 - 使用 php 标签获取完整 URL -->
{eyou:php}
  $queryString = $_SERVER['QUERY_STRING'];
  $baseUrl = '?'.$queryString;
  // 移除旧的 orderby/orderway 参数
  parse_str($queryString, $params);
  unset($params['orderby']);
  unset($params['orderway']);
  $baseUrl = '?'.http_build_query($params);
{/eyou:php}
<div class="filter-links">
  <a href="{$baseUrl}&orderby=add_time" class="filter-link">最新</a>
  <a href="{$baseUrl}&orderby=sales_num" class="filter-link">销量</a>
  <a href="{$baseUrl}&orderby=click" class="filter-link">热门</a>
  <a href="{$baseUrl}&orderby=users_price&orderway=asc" class="filter-link">价格</a>
</div>
```

> 或者在前端 JS 中处理（参考 lists_product.htm 第 393-416 行的现有实现）

## 四、搜索表单不应放在导航栏

搜索功能应在首页或列表页单独展示，不应放在导航栏中。

### 正确做法

使用 `{eyou:searchform}` 标签在首页或列表页合适位置放置搜索表单。

```html
{eyou:searchform type='sonself' id='field'}
  <form method="get" action="{$field.action}">
    <input type="text" name="keywords" placeholder="请输入关键词" />
    <input type="submit" value="搜索" />
    {$field.hidden}
  </form>
{/eyou:searchform}
```

搜索结果会自动使用对应模型模板（如 `lists_search_2.htm` 表示产品搜索结果）。

> 详见 [tags/searchform.md](tags/searchform.md)

## 六、sppurchase 只能有一个且必须包含三个必要元素

### 问题

`{eyou:sppurchase}` 标签在**一个产品页面中只能使用一次**，且内部必须包含三个必要元素：价格、数量、购买按钮。

### 错误写法

```html
<!-- 错误：出现两个 sppurchase 标签 -->
<div class="product-price-box">
  {eyou:sppurchase id='field'}
  <span>¥{$field.users_price}</span>
  <span>¥{$field.old_price}</span>
  {$field.hidden}
  {/eyou:sppurchase}
</div>

<div class="product-actions">
  {eyou:sppurchase id='field'}
  <button {$field.BuyNow}>立即购买</button>
  <button {$field.ShopAddCart}>加入购物车</button>
  {$field.hidden}
  {/eyou:sppurchase}
</div>
```

### 正确写法

```html
<!-- 正确：只有一个 sppurchase，包含全部必要元素 -->
<div class="product-price-box">
  {eyou:sppurchase id='field'}

  <!-- 1. 价格 -->
  <div class="product-price">
    <span class="product-price-unit">¥</span>{$field.users_price}
    <span class="product-price-original">¥{$field.old_price}</span>
  </div>

  <!-- 2. 数量 -->
  <div class="product-quantity">
    <button {$field.ReduceQuantity}>-</button>
    <input type="text" {$field.UpdateQuantity} value="1">
    <button {$field.IncreaseQuantity}>+</button>
  </div>

  <!-- 3. 购买按钮 -->
  <div class="product-actions">
    <button {$field.BuyNow}>立即购买</button>
    <button {$field.ShopAddCart}>加入购物车</button>
  </div>

  <!-- 必须：隐藏域 -->
  {$field.hidden}
  {/eyou:sppurchase}
</div>
```

### 三个必要元素

| 元素 | 变量/属性 | 说明 |
| -- | -- | -- |
| 价格 | `{$field.users_price}`、`{$field.old_price}` | 会员价、原价/划线价 |
| 数量 | `{$field.ReduceQuantity}`、`{$field.UpdateQuantity}`、`{$field.IncreaseQuantity}` | 减号按钮、数量输入框、加号按钮 |
| 购买按钮 | `{$field.BuyNow}`、`{$field.ShopAddCart}` | 立即购买、加购按钮 |

> 详见 [tags/sppurchase.md](tags/sppurchase.md)
