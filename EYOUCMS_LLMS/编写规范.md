# 编写规范

> 本规范用于指导 EYOUCMS 前端模板的编码，聚焦于 HTML/CSS/JS 的具体实现约束。
>
> ⚠️ **常见问题**：[问题记录.md](问题记录.md)

---

## 一、目录结构

> 强制要求 `.htm` 扩展名。

```
html/
├── header.html          # 头部框架（已存在，勿修改）
├── footer.htm           # 底部片段
├── nav.htm              # 导航栏片段
├── index.htm            # 首页
├── lists_article.htm    # 文章列表页
├── lists_article_small.htm   # 窄型文章列表页
├── lists_search.htm     # 搜索列表页
├── lists_single.htm     # 单页面
├── lists_product.htm    # 产品列表页
├── view_article.htm     # 文章详情页
└── view_product.htm     # 产品详情页
```

---

## 二、页面结构模板

每个页面必须严格遵循以下结构：

```html
<!DOCTYPE html>
<html>
<head>
  {eyou:include file="header.htm" /}
  <style>
    :root {
      /* 所有 CSS 变量在此定义 */
      --color-primary: #e84393;
      --color-secondary: #ff6b35;
      --color-surface: #ffffff;
      --color-on-surface: #2d3436;
      --color-border: #f0f0f0;
    }
    /* 自定义样式（禁止 @layer、@apply、@theme） */
  </style>
</head>
<body>
  <!-- nav -->
  {eyou:include file="nav.htm" /}
  <!-- nav -->

  <!-- [页面主内容区] -->

  <!-- footer -->
  {eyou:include file="footer.htm" /}
  <!-- footer -->
  <script>
    (function () {
      // 当前页面逻辑
    })();
  </script>
</body>
</html>
```

> `header.html` 和 `nav.htm`、`footer.htm` 的注释行必须保留。

## 三、公共片段编写规范

### 3.1 nav.htm / footer.htm

- 禁止使用 `<style type="text/tailwindcss">`，但可直接使用 Tailwind 类名
- 允许使用普通 `<style>` 标签写少量私有样式
- JS 写在片段最下方，包裹 IIFE

```html
<style>
  /* 仅限本片段私有样式 */
</style>

<nav id="site-nav">
  <!-- {eyou:models} 导航结构 -->
</nav>

<script>
  (function () {
    // 当前片段脚本
  })();
</script>
```

## 四、编码规范

### 4.1 TailwindCSS 使用规范

由于使用 TailwindCSS Browser CDN（`lib/tailwind-browser.min.js`），有以下强制限制：

- 禁止使用 `@layer` 指令
- 禁止使用 `@apply` 指令
- 禁止使用 `<style type="text/tailwindcss">` 标签
- 所有自定义样式必须使用普通 `<style>` 标签

**正确写法**：

```html
<style>
  :root {
    --color-primary: #e84393;
    --color-surface: #ffffff;
    --color-on-surface: #2d3436;
  }
  .product-card {
    background: var(--color-surface);
    border-radius: 0.75rem;
    transition: transform 0.3s ease;
  }
  .product-card:hover {
    transform: translateY(-6px);
  }
</style>
```

**错误写法**（会导致样式不生效）：

```html
<!-- 禁止使用 @layer 和 @apply -->
<style type="text/tailwindcss">
@theme { --color-primary: #e84393; }
@layer components { .btn { @apply bg-primary text-white; } }
</style>
```

其他规则：
1. 样式优先使用 Tailwind 内置类名（如 `class="w-full h-12 bg-white"`）
2. CSS 变量用于自定义颜色，在 `:root` 中定义
3. 使用 Tailwind 兼容的数值（如 `0.75rem` 相当于 `text-xl`）

### 4.2 jQuery 使用规范

1. 必须包裹 IIFE，隔离作用域
2. 选择器必须带块级前缀（如 `#block-hero`、`.block-latest .card`）
3. 禁止跨块操作 DOM
4. JS 代码写在块的最下方

```javascript
(function () {
  $('#block-hero .btn').on('click', function () {
    // ...
  });
})();
```

### 4.3 Swiper 使用规范

1. 每个 Swiper 实例用唯一 ID
2. 所有配置写在 IIFE 中，不污染全局作用域

```javascript
(function () {
  new Swiper('#hero-swiper', {
    loop: true,
    autoplay: { delay: 5000, disableOnInteraction: false },
    navigation: {
      nextEl: '#hero-swiper .swiper-button-next',
      prevEl: '#hero-swiper .swiper-button-prev'
    }
  });
})();
```

### 4.4 HTML 模板规范

1. 缩进统一使用 2 空格
2. 属性值必须加引号
3. 标签必须闭合（自闭合标签使用斜线风格 `<img />`）
4. 禁止使用内联样式（`style=""`）和内联事件处理器（`onclick=""`）
5. 块级容器需加唯一 ID
6. 语义化标签优先：使用 `<header>`、`<nav>`、`<main>`、`<section>`、`<footer>` 等

```html
<!-- Good -->
<section id="block-hero" class="relative w-full h-screen">
  <h1 class="text-4xl font-bold">标题</h1>
</section>

<!-- Bad -->
<div class="relative w-full h-screen">
  <div class="text-4xl font-bold">标题</div>
</div>
```

### 4.5 EYOUCMS 标签使用规范

1. 标签所有属性值使用单引号包裹
2. 循环体内使用 `id='field'` 作为变量名
3. 列表数据需判断空状态：`{eyou:if condition="$field['title']"}`
4. 禁止输出原始空值
5. 分页标签必须放在 `<div class="pagination">` 容器内

```html
{eyou:list loop='10' titlelen='30' orderby='add_time' id='field'}
  {eyou:if condition="$field.title"}
  <a href='{$field.arcurl}'>{$field.title}</a>
  {/eyou:if}
{/eyou:list}

<div class="pagination">
  {eyou:pagelist listitem='index,pre,pageno,next,end,info' listsize='2' /}
</div>
```

### 4.6 Logo 规范

Logo 必须使用后台配置，禁止硬编码。

```html
<!-- 必须使用全局变量调取 -->
<a href="/">
  <img src='{eyou:global name="web_logo" /}' alt="网站Logo" />
</a>
```

---

### 4.7 图片与静态资源规范

1. 图片必须指定 `alt` 属性，禁止留空
2. 禁止使用占位图服务
3. 图片路径使用相对路径，不得硬编码完整 URL
4. 静态资源路径使用 `/{$template_style}/` 前缀
5. 禁止硬编码人名、地名、品牌名

```html
<!-- Good -->
<img src='/{$template_style}/images/default.jpg' alt='{$field.typename}' />

<!-- Bad -->
<img src='https://example.com/logo.png' alt='' />
```

### 4.8 Font Awesome 图标规范

1. 禁止使用 emoji，统一使用 Font Awesome 图标
2. 优先使用 Solid 风格图标（`fas`）
3. 图标须有语义化用途，不得纯装饰性使用

```html
<!-- Good -->
<i class="fa-solid fa-search"></i>
<i class="fa-solid fa-phone"></i>

<!-- Bad -->
<i class="fa-solid fa-star" style="color: gold;"></i>
```

## 五、禁止行为总表

| 禁止项 |
| -- |
| 修改 header.html 结构 |
| 渲染 "undefined"、"null"、"nil" 等原始值 |
| 使用 Lorem ipsum 等占位文本 |
| 硬编码人名、地名、品牌名等真实信息 |
| 在 nav.htm / footer.htm 中使用 `<style type="text/tailwindcss">` |
| 跨块操作 DOM |
| 使用内联样式 `style=""` |
| 使用内联事件处理器 `onclick=""` |
| 使用 emoji（统一用 Font Awesome 图标） |
| 使用 SVG（内联或外部引用均不允许） |
| 在 header.html 中写具体样式 |
